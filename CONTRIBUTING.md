# Contributing to Community Tips

Thank you for helping grow Digging Code! Your contributions make a real difference.

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button on the top-right of this repo to create your own copy.

### 2. Create a New Branch

Create a branch for your tip to keep changes organized:

```bash
git checkout -b add-tip-[topic-name]
```

### 3. Add Your Tip

- Create a new Markdown file for your tip with the topic name, e.g., `eloquent.md`.
- Keep your tip clear and concise:
  - **Brief headline**
  - Optional example or code snippet

### 4. Commit Your Changes

```bash
git add .
git commit -m "Add tip about [topic]"
```

### 5. Push Your Branch

```bash
git push origin add-my-tip
```

### 6. Open a Pull Request

- Go to your fork on GitHub.
- Click **Compare & pull request**.
- Target the `main` branch of this repository.
- Add a short description of your tip and the topic it belongs to.
- Submit the PR for review.

---

## Guidelines

- Make sure your tip is **relevant** and **accurate**.
- Check existing topics to avoid duplicates.

---

## Review Process

All PRs will be **reviewed before merging**.
Contributors will be credited in the commit message and on the tip itself if it is published on the blog.
